<interfaces>
Pattern established in plan 01-03 (shared libraries):
  Replace: #if DEBUG ... #else ... #endif
  With:    if (builder.Environment.IsDevelopment()) { ... } else { ... }

Each persistence DI file has the same structure:
  #if DEBUG
    builder.AddNpgsqlDbContext<TDbContext>(connectionName: AppResources.XxxDb, ...);
    services.AddScoped<IUnitOfWork, UnitOfWork<TDbContext>>();
  #else
    var connectionString = GetConnectionString(services);
    services.AddDatabase<TDbContext>(connectionString, ...);
    static string GetConnectionString(...) { return services.AddValidateOptions<PostgresCpOptions>().ConnectionString; }
  #endif

Note: In the DEBUG branch, AddScoped<IUnitOfWork> is inside the #if. In the else branch it is absent.
In Shops and Moderation the UnitOfWork registration is inside #if DEBUG.
In Account it is also inside #if DEBUG.
In MediaService/Program.cs the UnitOfWork registration is OUTSIDE both branches (line 50: services.AddScoped<IUnitOfWork, UnitOfWork<MediaDbContext>>()).

After the conversion, IUnitOfWork registration must be preserved correctly:
- For Account, Shops, Moderation: move AddScoped<IUnitOfWork, ...> outside the if/else block (it is needed in both paths).
- For MediaService: already outside — no change needed there.

IHostEnvironment is available as builder.Environment (type IWebHostEnvironment, which extends IHostEnvironment).
Microsoft.Extensions.Hosting is already imported in the existing files (IHostEnvironment referenced for the existing using Microsoft.Extensions.Hosting line).
</interfaces>
