Output: Modified AuthService.cs, modified UserNameChangedEventHandler.cs.

From CoffeePeek.Account.Domain/Entities/UserAggregate/User.cs:

AddSession already enforces the MaxActiveSessions limit internally:
  public void AddSession(string token, TimeSpan ttl, string device, string ip)
  {
      if (_refreshTokens.Count(t => t.IsActive) >= BusinessConstants.MaxActiveSessions)
      {
          _refreshTokens.Where(t => t.IsActive).OrderBy(t => t.CreatedAtUtc).First().Revoke();
      }
      _refreshTokens.Add(new RefreshToken(Id, token, ttl, device, ip));
  }

RevokeAllSessions() is still used legitimately in RotateRefreshToken's security-breach path — do NOT remove the method itself.
