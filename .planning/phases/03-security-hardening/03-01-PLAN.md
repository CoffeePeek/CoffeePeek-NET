<objective>
Fix Sentry PII configuration in committed appsettings.json files across three services.

Purpose: Three service configs have `SendDefaultPii: true` and `MaxRequestBodySize: "Always"`. If a Sentry DSN is ever populated (e.g., during incident investigation), request bodies — which include plaintext passwords in login flows — will be transmitted to Sentry. This is a latent data-exfiltration risk that must be closed in committed config, not left to runtime env vars.

Output: Three appsettings.json files with `"SendDefaultPii": false` and `"MaxRequestBodySize": "None"`. All other Sentry keys and all non-Sentry config untouched.
</objective>
