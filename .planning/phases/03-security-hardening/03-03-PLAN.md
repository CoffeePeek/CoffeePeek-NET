<objective>
Fix rate limiting partition key to use X-Forwarded-For and add [AllowAnonymous] to MapController.

Purpose: The GlobalLimiter partitions by context.Connection.RemoteIpAddress which, behind Railway's proxy layer, is always the proxy's internal IP. All clients share one rate limit bucket, making per-client throttling completely ineffective. The fix reads X-Forwarded-For (Railway injects the real client IP as the leftmost entry), with X-Real-IP and RemoteIpAddress as fallbacks. MapController has no [AllowAnonymous] attribute, so the OpenAPI spec does not correctly signal that this endpoint is intentionally public. The BearerSecurityTransformer already checks for IAllowAnonymous metadata — adding the attribute is sufficient.

Output: RateLimitingExtensions.cs with proxy-aware partition key; MapController.cs with [AllowAnonymous] attribute and its using import.
</objective>
