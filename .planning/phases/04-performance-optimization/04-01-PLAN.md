<objective>
Eliminate two query performance bottlenecks in CoffeeShopQueries.cs:

1. PERF-01: The MinRating filter currently generates a correlated subquery (a `FirstOrDefault()` subselect evaluated per outer row). Rewrite it as a LINQ Join to a pre-aggregated GroupBy subquery so EF Core emits a single SQL INNER JOIN against a derived table.

2. PERF-03: GetUserFavoriteCoffeeShops unnecessarily calls `.Include(x => x.CoffeeShop)` before `ProjectToType<CoffeeShopDetailsDto>()`. Mapster's projection generates its own JOIN; the Include forces EF Core to do redundant eager-loading work on tracked entities.

Purpose: Both fixes are surgical — each removes the bottleneck without changing observable behavior.
Output: Modified CoffeeShopQueries.cs with two rewritten methods.
</objective>

  <action>
    In the `Search` method of `CoffeeShopQueries`, replace the entire `if (request.MinRating.HasValue)` block with a JOIN-based form.

    Remove the current block:
    ```
    if (request.MinRating.HasValue)
    {
        query = query.Where(s => context.Reviews
            .Where(r => r.CoffeeShopId == s.Id && !r.IsSoftDelete)
            .GroupBy(r => r.CoffeeShopId)
            .Select(g => g.Average(r => r.Rating.AverageRating))
            .FirstOrDefault() >= request.MinRating.Value);
    }
    ```

    Replace with (per PERF-01):
    - Declare `var minRating = request.MinRating.Value;` (captured outside the query to avoid closure issues)
    - Declare `var ratingSubquery = context.Reviews.Where(r => !r.IsSoftDelete).GroupBy(r => r.CoffeeShopId).Select(g => new { CoffeeShopId = g.Key, Avg = g.Average(r => r.Rating.AverageRating) });`
    - Rewrite query as: `query = query.Join(ratingSubquery, s => s.Id, r => r.CoffeeShopId, (s, r) => new { Shop = s, r.Avg }).Where(x => x.Avg >= minRating).Select(x => x.Shop);`

    The LEFT JOIN variant (to include shops with no reviews) is NOT needed here: the original `FirstOrDefault()` returned 0.0 for shops with no reviews, which would fail any `>= minRating` filter. The INNER JOIN preserves that behavior: shops with no reviews are excluded when a MinRating filter is active. Add an inline comment: `// INNER JOIN: shops with no reviews are excluded when MinRating filter is active — consistent with previous behavior`

    Do NOT change any other part of the method. The `totalCount` and pagination queries remain unchanged.
  </action>
