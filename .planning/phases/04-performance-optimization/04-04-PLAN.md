<objective>
Replace the full sequential scan search predicate with index-accelerated ILIKE and add the required PostgreSQL GIN trigram indexes (PERF-02).

Three coordinated changes:
1. Query rewrite in `CoffeeShopQueries.Search()`: replace `s.Name.ToLower().Contains(term)` with `EF.Functions.ILike(s.Name, $"%{term}%")` and same for `s.Location.Address`.
2. Extension + index config: declare `pg_trgm` extension in `ShopsDbContext.OnModelCreating` and two GIN indexes in `CoffeeShopConfiguration` — one on `Name` (root entity builder), one on `Location.Address` (inside the `OwnsOne` closure).
3. EF Core migration: run `make mig-shops n=AddTrigramIndexes` then `make up-shops` to generate and apply the migration.

This plan runs in Wave 2 (after Wave 1 plans complete and tests are green) because it requires a checkpoint for migration verification.

Purpose: `LOWER().Contains()` generates `LOWER(column) LIKE '%term%'` — un-indexable. `EF.Functions.ILike` maps to PostgreSQL `ILIKE`, which the GIN trigram index can accelerate.
Output: Modified query + configuration + new migration files.
</objective>

    Two indexes to add:

    1. Name index — add after the existing `builder.Property(x => x.Description)...` line (on the root `builder`):
