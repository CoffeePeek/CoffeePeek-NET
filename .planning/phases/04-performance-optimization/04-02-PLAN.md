<objective>
Eliminate the N+1 UPDATE pattern in UserNameChangedHandler (PERF-05).

Current behavior: loads all Review entities for the user into memory (`GetByUserId`), iterates with `foreach review.UpdateUserName(...)`, then calls `SaveChangesAsync` — produces N individual UPDATE statements.

Target behavior: one `ExecuteUpdateAsync` call that issues a single `UPDATE Reviews SET UserName = @p WHERE UserId = @id AND UserName != @p` directly on the database.

Purpose: UserName changes can affect many reviews; loading all into memory is O(N) in memory and round-trips.
Output: Rewritten UserNameChangedEventHandler.cs with direct DbContext injection.
</objective>
