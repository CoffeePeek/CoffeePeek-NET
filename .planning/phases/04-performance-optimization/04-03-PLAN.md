<objective>
Fix the blocking Redis KEYS command in RedisService.RemoveByPattern (PERF-04).

Current behavior: `_server.Keys(pattern: pattern).ToArray()` — synchronous enumeration that may issue the blocking `KEYS` command, which can freeze the Redis server for the duration of the scan when the keyspace is large.

Target behavior: `await foreach` over `_server.KeysAsync(pattern: pattern, pageSize: 250)` — StackExchange.Redis internally uses the non-blocking `SCAN` cursor command when `pageSize` is supplied.

Purpose: KEYS is a known Redis anti-pattern in production. SCAN is the safe alternative for pattern-based key enumeration.
Output: Modified RemoveByPattern method in RedisService.cs.
</objective>
